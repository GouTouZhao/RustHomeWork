## 3. 项目背景与目标
**项目背景：**
随着网络技术的发展，人们对于高性能和高并发交流平台的需求日益增加。传统的单体架构系统在用户量增长时往往面临性能瓶颈。本项目以现有的 `blognew`（基于 Go 语言）项目为基础进行改进重构，全面采用内存安全且性能极高的 Rust 语言重写后端服务，同时剥离原有系统中繁杂的博客功能，专注于打造一个纯粹的、企业级的论坛交流平台。

**项目目标：**
- **功能目标**：实现完整的论坛发帖、浏览、评论、个人主页等核心功能。
- **技术目标**：
  - 将原有的 Go 后端微服务框架平滑迁移至 Rust 语言体系。
  - 维持原有的企业级微服务架构体系，包括使用 gRPC 进行服务间通信以及使用网关进行 API 路由转发。
  - 实现基于 Kubernetes (k8s) 的容器化高可用部署，支持通过 `docker push` 与 `k8s apply` 流程一键发布。
  - 前端基于 Vue 3 框架，构建响应式、轻量化的论坛用户界面。

## 4. 系统设计与实现思路
**1) 整体架构：**
项目采用了经典的前后端分离 + 后端微服务架构，并通过 Kubernetes 进行容器编排。
- **前端 (ForumFrontend)**：基于 Vue 3 + Vite 构建，页面包含首页/分区页 (`PartitionView`)、文章详情 (`ArticleView`)、发帖页面 (`PostView`) 与用户中心 (`ProfileView`)。前端通过 HTTP(S) 请求网关服务。
- **网关 (API Gateway)**：由 Rust (`Axum` 框架) 编写的 API 网关，负责跨域 (CORS) 处理、请求路由转发，并提供外部统一访问入口。
- **微服务层 (RPC 层)**：后端逻辑拆分为多个微服务模块：
  - `user`：用户微服务，处理注册、登录、身份鉴权等。
  - `bmanager`：管理微服务，处理文章列表、文章发布、分类及帖子管理。
  - `admin`：后台管理微服务。
  - `image` / `static_svc`：静态资源与图片服务。

**2) 核心数据结构与算法：**
- 服务间通信采用 **Protocol Buffers (protobuf)** 定义数据模型，确保数据交互强类型与高效率。
- 后端使用统一标准化的 Error 类型 (错误标准化)，方便全局拦截和日志输出。
- 引入 **Redis** 对热门帖子和用户信息进行缓存，降低 **MySQL** 数据库的直接查询压力。

**3) 部署环境设计：**
- 提供 `.dockerignore` 和 `Dockerfile` 进行轻量化镜像构建。
- `deployments/` 目录中维护了 Kubernetes 资源文件 (`.yaml` 格式)，结合 `start-k8s.bat` 等脚本，实现从代码编译、Docker 镜像打包到 k8s 中 Service/Deployment 部署的一键式 CI/CD 流程。

## 5. 实验结果
**项目已经实现的主要功能如下：**
1. **统一登录与鉴权**：通过网关完成用户登录验证与 Token 分发，保障接口调用安全。
2. **论坛专区浏览**：前端去除了冗余的博客模块，保留发帖相关业务，提供清晰的版块内容列表展示。
3. **内容发布与详情交互**：用户可进入发帖页发布新帖，其他用户可在详情页中查看具体内容，实现富文本支持。
4. **服务容器化与集群运行**：所有微服务均已 Docker 化，能够稳定运行于本地或云端的 Kubernetes 集群中。

## 6. 开源项目参考说明
**参考来源：**
本项目参考借鉴了开源项目 `blognew` (Go 语言后端) 以及 `FrontBlogNew` (Vue 3 前端)。

**引用方式：**
本项目的微服务拆分逻辑、Kubernetes 部署架构、数据库（MySQL/Redis/OSS）表结构及部分连接配置均参考了原开源项目的整体设计。

**本项目与原项目的区别、修改和改进之处：**
1. **技术栈的底层重构**：将后端的 Go 语言全面替换为 Rust 语言。利用 Rust 的所有权机制和零成本抽象特性，在不增加内存开销的前提下，极大地提升了微服务的内存安全性和高并发处理能力。项目全面引入了 `tokio` 和 `axum` 等 Rust 高性能生态库。
2. **功能剥离与聚焦**：去除了原项目中冗余的“个人博客”等分区模块，专注实现“论坛与发帖”功能。前端 UI 层面基于 Vue 3 进行了针对性的精简和修改，使界面更契合纯粹的论坛需求。
3. **架构原样保留与生态适配**：在保持了原有企业级微服务封装、调用逻辑、报错标准化等优秀设计的前提下，成功用 Rust 语言走通了整个 k8s 和 gRPC 流程，展现了良好的架构可移植性。
